# i18n — Design

Data: 2026-05-13
Status: aprovado para implementação

## Objetivo

Adicionar internacionalização à aplicação Planning Poker, com suporte inicial a três idiomas: Português (Brasil), Inglês (EUA) e Espanhol. O idioma é detectado automaticamente a partir do navegador no primeiro acesso, com fallback `pt-BR`, e pode ser trocado manualmente via seletor no header. A escolha do usuário persiste entre sessões.

Conteúdo gerado pelo usuário (nome da sala, nomes dos participantes, cartas customizadas) **não** é traduzido — somente o chrome da UI.

## Decisões

- **Biblioteca:** `vue-i18n@^11`, em modo Composition API (sem `legacy`).
- **Idiomas:** `pt-BR` (padrão), `en`, `es`.
- **Detecção:** `localStorage` → `navigator.language` → fallback `pt-BR`.
- **Persistência:** chave `pp.locale` em `localStorage`, gravada apenas quando o usuário escolhe explicitamente no seletor.
- **Seletor:** componente compacto no `AppHeader`, ao lado do toggle de dark mode.
- **Decks (`src/lib/decks.ts`):** rótulos e descrições passam a ser chaves de tradução (`labelKey`/`descKey`); valores das cartas (`1`, `2`, `M`, `GG`, etc.) permanecem inalterados para não afetar dados persistidos no Firestore.
- **Não-objetivos:** lazy-load de locales, formatação de número/data, RTL, tradução de conteúdo do usuário.

## Arquitetura

### Estrutura nova

```
src/
  i18n/
    index.ts              # createI18n + detecção inicial + singleton exportado
    locales/
      pt-BR.json
      en.json
      es.json
    types.ts              # MessageSchema (autocomplete de t())
  composables/
    useLocale.ts          # current, setLocale, supported
  components/ui/
    LocaleSwitcher.vue    # dropdown no AppHeader
```

### `src/i18n/index.ts`

Exporta:

```ts
export const SUPPORTED_LOCALES = ['pt-BR', 'en', 'es'] as const
export type Locale = (typeof SUPPORTED_LOCALES)[number]
export const DEFAULT_LOCALE: Locale = 'pt-BR'
export const i18n = createI18n<[MessageSchema], Locale>({ ... })
```

Detecção inicial (executada uma vez, no carregamento do módulo):

1. `localStorage.getItem('pp.locale')`. Se for valor presente em `SUPPORTED_LOCALES`, usa esse.
2. Senão, lê `navigator.language` e normaliza por prefixo:
   - `pt*` → `pt-BR`
   - `en*` → `en`
   - `es*` → `es`
   - outro → `DEFAULT_LOCALE`
3. **Não persiste** nada no passo de detecção automática — só persiste quando o usuário trocar pelo seletor.

Após criar o `i18n`, ajusta `document.documentElement.lang` para o locale inicial.

### `src/composables/useLocale.ts`

```ts
export function useLocale() {
  const { locale } = useI18n()
  const current = computed(() => locale.value as Locale)
  function setLocale(next: Locale) {
    locale.value = next
    localStorage.setItem('pp.locale', next)
    document.documentElement.lang = next
  }
  return { current, setLocale, supported: SUPPORTED_LOCALES }
}
```

### `src/components/ui/LocaleSwitcher.vue`

- Trigger: botão compacto exibindo a sigla atual (`PT` / `EN` / `ES`) e um chevron.
- Click abre um dropdown listando os três idiomas com label nativa (`Português`, `English`, `Español`).
- Acessibilidade: `aria-haspopup="listbox"`, `aria-expanded`, fechamento por `Esc` e click-fora, foco devolvido ao trigger ao fechar.
- Mobile e desktop usam o mesmo dropdown.
- Posicionamento: dentro do `AppHeader`, imediatamente antes do toggle de dark mode.

### Wiring no `src/main.ts`

```ts
import { i18n } from './i18n'
// ...
app.use(pinia).use(router).use(i18n)
```

`app.use(i18n)` deve ocorrer antes de `app.mount('#app')`.

### Uso fora de SFC (composables, stores)

Importar o singleton e chamar `i18n.global.t(...)`. Isso vale para `roomStore`, `useEmojiBroadcast`, `useThinking`, `useToasts` e quaisquer outros pontos que produzam mensagens visíveis ao usuário.

## Conteúdo dos locales

`pt-BR.json` é a fonte; `en.json` e `es.json` espelham todas as chaves. Estrutura por namespace:

```
common.*    cancel, confirm, back, close, save
header.*    toggleDark, selectLanguage
home.*      heroTitle, heroSubtitle, ctaCreate, footer.madeBy
create.*    title, deckLabel, customDeck.*, howItWorks.*
room.*      join.*, reveal, reset, copyLink, removePlayer,
            thinking, results.* (average, mode, min, max,
            distribution, votesCount)
decks.*     fibonacci.{name,description}, tshirt.{name,description},
            modified.{name,description}
toasts.*    linkCopied, playerKicked, connectionLost
errors.*    roomNotFound, permissionDenied
```

Convenções:

- **Pluralização:** sintaxe nativa do vue-i18n (`"nenhum voto | {n} voto | {n} votos"`). Usada inicialmente em `room.results.votesCount`.
- **Interpolação:** sempre por slot nomeado (`{name}`, `{n}`), nunca concatenação.
- **Atributos HTML traduzíveis:** `<html lang>`, `aria-label` de botões só com ícone, `title` da página (via router meta + watcher do locale).

## Critérios de aceite

- [ ] Os três idiomas funcionam e cobrem 100% dos textos visíveis hoje.
- [ ] Seletor no `AppHeader` troca o idioma sem reload.
- [ ] Idioma escolhido persiste entre sessões via `localStorage` (`pp.locale`).
- [ ] Primeiro acesso usa `navigator.language` normalizado, com fallback `pt-BR`.
- [ ] `<html lang>` reflete o locale corrente.
- [ ] `bun run lint:types` passa.
- [ ] `bun run test` passa, incluindo o smoke de cobertura.
- [ ] `bun run test:integration` passa sem alterações nas regras do Firestore.
- [ ] Grep final por padrões PT-BR (`[áéíóúâêôãõç]`, palavras como "Cancelar", "Revelar") nos `.vue`/`.ts` de `src/` não encontra texto solto fora dos JSON de locale.

## Riscos

- **Texto solto esquecido em algum componente** — mitigação: grep final + revisão manual dos 16 arquivos listados.
- **Quebra do shape de `decks.ts`** — `CreateSessionView` e `roomStore` consomem o `lib`; ambos são atualizados na mesma PR.
- **Tradução EN/ES de termos de Scrum/Planning Poker** — usar termos consagrados (`Reveal`, `New round`, `Story points`); qualquer escolha duvidosa fica comentada na PR para revisão.
- **Bundle size** — +~13kB gzip (vue-i18n) + ~2kB (JSONs). Aceitável.

## Fora de escopo

- Tradução de conteúdo gerado pelo usuário (nomes de sala, participantes, cartas customizadas).
- Lazy-load de locales (só faz sentido com mais idiomas).
- Formatação de número/data por locale.
- RTL.
